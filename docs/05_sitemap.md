# サイトマップ - 人材エージェントマイページシステム

## 1. 前提情報

### サイトの目的
人材エージェントと応募者が情報を共有・管理できるWebシステム。応募者は自身の情報管理と求人閲覧、エージェントは応募者管理と求人管理を行う。

### 想定ユーザー
- **応募者**: 転職・就職活動中の個人
- **エージェント**: 人材エージェント会社の社員
- **年齢層**: 20-50代
- **デバイス**: PC、タブレット、スマートフォン
- **利用目的**: 
  - 応募者: 個人情報管理、求人情報閲覧、書類提出、応募状況確認
  - エージェント: 応募者情報管理、求人管理、ステータス管理、商談記録

### 技術構成
- **フロントエンド**: React + TypeScript + Bootstrap（レスポンシブデザイン）
- **バックエンド**: Node.js / Python
- **認証**: セッションベース認証
- **デプロイ**: 自社サーバー（オンプレミス）

3. **個人情報閲覧・編集画面** - プロフィール管理

7. **応募状況確認画面** - 応募中の求人ステータス
8. **通知一覧** - システム内通知の確認
9. **パスワード変更画面** - パスワード設定変更

### エージェント管理画面
10. **ログイン画面** - エージェントのログイン
11. **ダッシュボード** - 全体統計と最新情報

13. **応募者詳細画面** - 応募者の詳細情報と操作
14. **全文検索画面** - 応募者情報の横断検索
15. **求人一覧** - 求人情報の管理
16. **求人詳細画面** - 求人の詳細と応募者紐付け
17. **求人作成・編集画面** - 求人情報の登録・更新
18. **商談メモ一覧** - 応募者ごとのメモ管理
19. **商談メモ作成・編集画面** - メモの作成・編集
20. **活動履歴画面** - システムの操作履歴
21. **データエクスポート画面** - CSV出力
22. **パスワード変更画面** - パスワード設定変更

### 共通・システムページ
23. **404エラーページ** - ページが見つからない
24. **403エラーページ** - アクセス権限なし
25. **500エラーページ** - サーバーエラー

## 3. ページのグループ分け

### グループA: 応募者マイページ（認証必須、role: applicant）
応募者専用の画面。自分の情報のみアクセス可能。

- ダッシュボード
- 個人情報閲覧・編集画面
- 求人情報一覧
- 求人詳細画面
- 書類管理画面
- 応募状況確認画面
- 通知一覧
- パスワード変更画面

### グループB: エージェント管理画面（認証必須、role: agent）
エージェント専用の画面。全応募者・求人にアクセス可能。

- ダッシュボード
- 応募者一覧
- 応募者詳細画面
- 全文検索画面
- 求人一覧
- 求人詳細画面
- 求人作成・編集画面
- 商談メモ一覧
- 商談メモ作成・編集画面
- 活動履歴画面
- データエクスポート画面
- パスワード変更画面

### グループC: 認証（認証不要）
ログイン前の画面。

- ログイン画面（応募者・エージェント共通）

### グループD: エラー・システム
- 404エラーページ
- 403エラーページ
- 500エラーページ

## 4. 階層構造の決定

### 第一階層
- **ルート (/)** - ログイン後、役割に応じてリダイレクト
  - 応募者 → /applicant/dashboard
  - エージェント → /agent/dashboard

### 第二階層（応募者）
- **/applicant** - 応募者マイページ
  - **/applicant/dashboard** - ダッシュボード
  - **/applicant/profile** - 個人情報
  - **/applicant/jobs** - 求人一覧
  - **/applicant/files** - 書類管理
  - **/applicant/applications** - 応募状況
  - **/applicant/notifications** - 通知一覧
  - **/applicant/settings** - 設定

### 第三階層（応募者）
- **/applicant/jobs/:id** - 求人詳細
- **/applicant/settings/password** - パスワード変更

### 第二階層（エージェント）
- **/agent** - エージェント管理画面
  - **/agent/dashboard** - ダッシュボード
  - **/agent/applicants** - 応募者一覧
  - **/agent/search** - 全文検索
  - **/agent/jobs** - 求人一覧
  - **/agent/activity-logs** - 活動履歴
  - **/agent/export** - データエクスポート
  - **/agent/settings** - 設定

### 第三階層（エージェント）
- **/agent/applicants/:id** - 応募者詳細
- **/agent/applicants/:id/memos** - 商談メモ一覧
- **/agent/jobs/:id** - 求人詳細
- **/agent/jobs/new** - 求人作成
- **/agent/jobs/:id/edit** - 求人編集
- **/agent/memos/:id/edit** - 商談メモ編集
- **/agent/settings/password** - パスワード変更

### 認証・エラー
- **/login** - ログイン画面
- **/404** - 404エラーページ
- **/403** - 403エラーページ
- **/500** - 500エラーページ

## 7. 画面遷移図

### 主要な画面遷移フロー

#### 応募者側
```
[ログイン (/login)]
    ↓ ログイン成功
[ダッシュボード (/applicant/dashboard)]
    ↓
    ├→ [個人情報 (/applicant/profile)] → 編集・保存 → ダッシュボード
    ├→ [求人情報一覧 (/applicant/jobs)]
    │   └→ [求人詳細 (/applicant/jobs/:id)] → ダウンロード
    ├→ [書類管理 (/applicant/files)] → アップロード
    ├→ [応募状況 (/applicant/applications)]
    ├→ [通知一覧 (/applicant/notifications)] → 既読
    └→ [設定 (/applicant/settings)]
        └→ [パスワード変更 (/applicant/settings/password)]
```

#### エージェント側
```
[ログイン (/login)]
    ↓ ログイン成功
[ダッシュボード (/agent/dashboard)]
    ↓
    ├→ [応募者一覧 (/agent/applicants)]
    │   └→ [応募者詳細 (/agent/applicants/:id)]
    │       ├→ ステータス更新
    │       ├→ [商談メモ一覧 (/agent/applicants/:id/memos)]
    │       │   └→ [メモ編集 (/agent/memos/:id/edit)]
    │       └→ 求人紐付け
    │
    ├→ [全文検索 (/agent/search)]
    │   └→ 検索結果 → 応募者詳細
    │
    ├→ [求人一覧 (/agent/jobs)]
    │   ├→ [求人作成 (/agent/jobs/new)] → 保存 → 求人一覧
    │   └→ [求人詳細 (/agent/jobs/:id)]
    │       ├→ [求人編集 (/agent/jobs/:id/edit)] → 更新 → 求人詳細
    │       └→ 応募者紐付け
    │
    ├→ [活動履歴 (/agent/activity-logs)]
    ├→ [データエクスポート (/agent/export)] → CSV出力
    └→ [設定 (/agent/settings)]
        └→ [パスワード変更 (/agent/settings/password)]
```

### 詳細な遷移パターン

```
1. 初回ログインフロー
   /login → 認証 → /applicant/dashboard または /agent/dashboard

2. 応募者の求人閲覧フロー
   /applicant/dashboard → /applicant/jobs → /applicant/jobs/:id → ダウンロード

3. 応募者のプロフィール更新フロー
   /applicant/dashboard → /applicant/profile → 編集 → 保存 → /applicant/dashboard

4. エージェントの応募者検索フロー
   /agent/dashboard → /agent/applicants → フィルタリング → /agent/applicants/:id

5. エージェントの全文検索フロー
   /agent/dashboard → /agent/search → 検索 → 結果一覧 → /agent/applicants/:id

6. エージェントの求人作成フロー
   /agent/dashboard → /agent/jobs → /agent/jobs/new → 保存 → /agent/jobs

7. エージェントのステータス更新フロー
   /agent/applicants/:id → ステータス変更 → 保存 → 応募者に通知送信

8. エージェントの商談メモ作成フロー
   /agent/applicants/:id → /agent/applicants/:id/memos → 新規作成 → 保存

9. エラーフロー
   存在しないURL → /404
   権限のないページ → /403
   サーバーエラー → /500
```

## 9. レスポンシブデザイン対応

### ブレークポイント
- **モバイル**: 375px - 767px
- **タブレット**: 768px - 1023px
- **デスクトップ**: 1024px以上

### デバイス別のUI調整

#### モバイル（スマートフォン）
- ハンバーガーメニューによるナビゲーション
- テーブルはカード表示に変更
- フォームは縦積みレイアウト
- ファイルアップロードはタップ操作
- フィルターは折りたたみ可能

#### タブレット
- サイドバーナビゲーション（折りたたみ可能）
- テーブルは横スクロール対応
- 2カラムレイアウト

#### デスクトップ
- サイドバーナビゲーション（常時表示）
- 3カラムレイアウト
- ホバーエフェクト

## 評価／改善点

### 成果物の評価視点
- **ナビゲーション**: 直感的で迷わない構造か
- **URL設計**: RESTfulで分かりやすいか
- **画面遷移**: スムーズで自然な流れか
- **レスポンシブ**: 全デバイスで使いやすいか
- **アクセシビリティ**: 障がい者にも使いやすいか

### 今後の検討事項
- **パンくずリスト**: 現在位置の明確化
- **検索履歴**: 過去の検索条件の保存
- **お気に入り機能**: よく使う応募者・求人の登録
- **一括操作**: 複数の応募者に対する一括ステータス変更
- **カスタムビュー**: ユーザーごとの表示設定保存
